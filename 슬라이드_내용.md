## Slide 3 — 앱 소개

**제목**: 도서 SNS 앱이란?

**앱 핵심 구조**
- **프로필**: 내 인생책 최대 9권을 공개 전시하는 공간
- **보관함(Archive)**: 프로필 외에 더 많은 책을 무제한으로 쌓아두는 개인 서재

**온보딩**: 가입 시 인생책 3권을 **Profile에 필수 등록**

**사용자 여정 (플로우)**
```
가입 → [필수] Profile에 인생책 3권 등록
              ↓
       [선택] 보관함에 추가 책 저장  /  리뷰 작성
                    ↕
          다른 사람 프로필 방문 / 팔로우
```

- 오늘 분석에서 핵심 변수: 보관함(Archive) · 리뷰 · 팔로우 · 방문
- **보관함 사용과 리뷰는 모두 선택 사항** — 가입 이후 아무도 강제하지 않는 행동

## Slide 11 — 휴면 예측

**제목**: ⑤ 휴면 예측: 7일 만에 알 수 있는가?

**방법 선택 근거**
- Logistic Regression 최종 선택 — 로그 변환 후 AUC 최고(0.648), SHAP LinearExplainer로 해석 용이
- **피처 기간 = 가입 후 7일**: 온보딩 골든 윈도우 — 이 기간에 예측 가능해야 사용자가 앱을 쓰는 동안 조기 개입 가능

**이미지**: `fig_03_roc_pr.png` + `fig_03_shap.png`

| 모델 | AUC |
|------|-----|
| Logistic Regression (피처 엔지니어링) | **0.648** |
| Random Forest | 0.624 |
| XGBoost | 0.631 |

**SHAP 중요도 Top 3**
1. log_archived (아카이브 수)
2. log_books (책 수)
3. log_visits (방문 수)

**AUC 0.648이 낮은 이유**
- 클러스터링에서 확인했듯 86%가 첫 7일에 아카이브·리뷰를 하지 않음
- 모델 입장에서 대부분의 사용자가 피처 값 ≈ 0 → 활성/휴면 구분 불가
- **즉, AUC가 낮은 이유 = Aha Moment를 경험하는 사람 자체가 드물기 때문**
- 모델 실패가 아니라, 문제의 본질을 가리키는 신호

## Slide 17 — Aha Moment

**6개 분석이 하나를 가리킨다**

# Aha Moment

**"첫 주에 리뷰를 한 번이라도 작성한 사용자"**

> 아카이브는 가입 시 3권 필수 — 이미 모두가 경험한다.  
> 그럼에도 86%가 떠났다.  
> **진짜 분기점은 강제되지 않은 행동 — 리뷰다.**

근거:
- Cox: 리뷰 HR **0.715** (이탈 위험 28.5% 감소, 전체 변수 중 최강)
- Cox: 보관함 사용 HR 0.744 — 보관함을 쓴 21.2%가 훨씬 오래 생존
- SHAP: log_archived 중요도 1위 — 첫 7일 안에 보관함을 쓴 사람이 이탈 위험 낮음
- 클러스터링: 86% '조용한 다수'는 profile 3권 등록 이후 아무것도 안 함
- 이탈 피드백: 19% "독서 기록용으로 쓰기 부적합" = 기록(리뷰·보관함) 경험 부재
